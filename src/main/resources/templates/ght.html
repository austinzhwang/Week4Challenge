Workflow for Teams

  • Anything in the master branch is deployable
  • To work on something new, issue a pull request then create a descriptively named branch off of master (ie: new-oauth2-scopes)
  • Commit to that branch locally and regularly push your work to the same named branch on the server
  • When you need feedback or help, or you think the branch is ready for merging, then pull
  • After you have reviewed and tested the feature, you can merge it into master
  • Once it is merged and pushed to ‘master’, you can and should deploy immediately

Setting the URL for a team:
git remote add origin "https://'yourName'@github.com/'RepositoryOwnerName'/repository.git"

  1. When working on a team, one person should create the repository online and add the other team members as collaborators
  2. That same person should also create the project directory
  3. Next add a readme file:
  4. mkdir ProjectDir
  5. echo "#This is the ProjectName >>readme.md
  6. echo "The team members are John, George, Paul and Ringo" >>readme.md
  7. git commit -am "initial commit"
  8. git push -u origin master
